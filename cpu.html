A memória principal reúne todas as memórias que o processador consegue acessar para realizar o endereçamento e armazenar as instruções e dados que são executados pela cpu
O bus de endereço, tembém chamado de barramento de endereço é usado para especificar um endereço físico. Quando um processador ou dispositivo habilitado para DMA precisa ler ou escrever em um local de memória, ele especifica esse local de memória no barramento de endereço
O bus de dados, também chamado de barramento de dados, transporta informações entre a cpu, memória e outros periféricos, portanto é o responsável pela comunicação entre os componentes do sistema de forma eficiente e rápida
O bus de controle, também chamado de barramento de controle, controla as operações entre a cpu, memória e outros periféricos enviando sinais para executar as operações e sincronizando elas, fazendo com que sejam feitas no momento correto e eficientemente
A função do registrador 0 pode variar dependendo da cpu, mas em geral ele é utilizado para armazenar o valor 0, que é utilizado em operações matemáticas ou controle de fluxo, mas também pode ser utilizado como um registrador qualquer
O papel exato de um registrador 1 depende muito da cpu trabalhada, mas é normalmente utilizado como um armazenamento temporário geral, guardando resultados de operações, cálculos ou valores gerais
O registrador 2 é normalmente utilizado para guardar informações temporárias durante execução de operações, mas também pode armazenar operandos ou resultados para auxiliar em operações de controle
O registrador 3 normalmente tem a função de um registrador genérico, armazenando informações para execução de instruções, mas dependendo da cpu pode ter funções especializadas como contendo valores intermediários
A unidade lógica aritmética é responsável por realizar as operações matemáticas e lógicas em uma cpu, processando as informações nos registradores de acordo com as instruções dadas
O registrador Z em uma CPU geralmente se refere ao Zero Flag, que é um bit no registrador de status. Ele fica com o valor 1 quando o resultado de uma operação aritmética ou lógica é zero. Isso é usado para controlar o fluxo do programa
A unidade de controle é responsável por interpretar e coordenar a execução das instruções do processador, ordenando as operações e o fluxo de dados na ordem correta para que o sistema funcione de forma eficiente
A unidade de entrada e saída gerencia a troca de dados entre o processador e dispositivos externos à cpu como o mouse, gerenciando interrupções e controlando os buffers de memória, sendo responsável pela interação com o meio externo
O PC é um registrador na CPU que controla a sequência de execução das instruções de um programa. Ele aponta para o endereço da próxima instrução a ser executada e é atualizado após cada instrução. Isso garante que a CPU siga a ordem correta de execução, inclusive em casos de saltos ou loops
O RI pode ser um registrador de índice, que facilita o acesso a dados em tabelas ou sequências, ou um registrador de instrução, que armazena a instrução atual a ser executada para que possa sex executada de forma eficiente
O SP é um registrador que aponta para o topo da pilha de memória, usado para gerenciar dados temporários e durante chamadas de funções e interrupções, o SP ajusta-se para armazenar e recuperar essas informações, garantindo que o programa continue executando corretamente
O registrador Y é um tipo de registrador usado em algumas arquiteturas de processadores, geralmente para armazenar dados temporários ou como parte de operações, como contagem ou cálculos de deslocamento de memória. Seu uso específico varia dependendo da arquitetura do processador
